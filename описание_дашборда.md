# Дашборд «Кофе на Ozon» — архитектура

> Файл **частично автогенерируется**: раздел «Список графиков» пересобирается из `dashboard.html` скриптом `scripts/generate_dashboard_doc.py`.

## Файл

`dashboard.html` — одностраничное приложение на чистом HTML/CSS/JS (без сборки).  
ECharts v5.5.0 подключается с CDN. **Не работает из `file://`**, требуется HTTP-сервер.

## Поток данных

```
data.json  ──fetch()──►  parseJSONData(json)
                              │
                    ┌─────────┴──────────┐
                    │   allData (массив   │
                    │   плоских объектов) │
                    └─────────┬──────────┘
                              │
                    applyFilters()  ←── Filters Bar + localStorage
                              │
                    ┌─────────┴──────────┐
                    │   filteredData      │
                    └─────────┬──────────┘
                        ┌────┴────┐
                        │         │
                   renderKPI()  renderAllCharts()
                                     │
                              renderCharts()
                              ├── renderDemandDeficit()
                              ├── renderPricePerGramSales()
                              ├── ... (каждый график)
                              └── renderBuyoutPrice()
```

### Ключевые функции

| Функция | Назначение |
|---|---|
| `loadData()` | Загружает `data.json` через `fetch()`, вызывает `parseJSONData`, затем `initDashboard()`. При ошибке сети — читает из IndexedDB. |
| `parseJSONData(json)` | Преобразует `data.json` (массив массивов) в массив плоских объектов. Извлекает период из первой строки. Маппинг колонок по `NUMERIC_FIELDS`. |
| `initDashboard()` | Заполняет фильтры (бренды, категории, схемы), восстанавливает состояние из localStorage, применяет фильтры, рендерит KPI и графики. |
| `applyFilters()` | Фильтрует `allData` → `filteredData`, обновляет KPI и все графики. |
| `filterData(data, f)` | Чистая функция фильтрации по поиску, бренду, категории, схеме, Super, цене, выручке, выкупу, рекламе. |
| `renderKPI()` | Вычисляет и рендерит 8 базовых + 9 продвинутых KPI на основе `filteredData`. |
| `renderCharts()` | Вызывает все функции рендера графиков по порядку. |
| `createChart(id, option)` | Создаёт или обновляет ECharts инстанс. Ленивая инициализация. |
| `handleChartClick(id, params)` | Cross-filtering: клик по категории/бренду заполняет соответствующий фильтр. |
| `renderSkuDetail(row)` | Рендерит все блоки SKU-таба: заголовок, радар, графики сравнения, таблицы. |

## Система графиков (CHART_DEFS)

Графики регистрируются в массиве `CHART_DEFS` (конец `dashboard.html`, перед `buildUI()`).  
Каждый элемент:

```javascript
{
  id: 'ch-revenue-cat',              // ID для DOM-элемента и ECharts
  title: 'Выручка по категориям кофе', // Заголовок карточки
  desc: 'Какой тип кофе…',           // Подзаголовок (1 строка)
  help: {                            // Справка для модалки
    shows: 'Столбчатая диаграмма…',   // Что показывает
    how: 'Наведите курсор…',          // Как пользоваться
    why: 'Показывает…'               // Зачем нужен
  }
}
```

**Важно**: `CHART_DEFS` определяет только UI-обёртку (карточку с заголовком, справкой, кнопками).  
Логика рендера — отдельные функции, вызываемые из `renderCharts()`.  
При добавлении нового графика нужно:
1. Написать функцию `renderИмя(d)` 
2. Вызвать её в `renderCharts()`
3. Добавить элемент в `CHART_DEFS`

### 3 секции графиков

Разделение задаётся в `buildUI()` массивом `sections`:
```javascript
const sections = [
  { firstId: 'ch-demand-deficit', title: 'Приоритетная аналитика', … },
  { firstId: 'ch-visits-conv',    title: 'Эффективность карточки товара', … },
  { firstId: 'ch-revenue-cat',    title: 'Детальная аналитика', … }
];
```
Первый график с `id === firstId` начинает новую секцию.

## Фильтры

Фильтры сохраняются в `localStorage` под ключом `coffeeDashboardFilters` и восстанавливаются при загрузке.

**Cross-filtering**: клик по графику категории (`ch-revenue-cat`, `ch-daily-sales`, etc.) заполняет `#filterCategory`.  
Клик по графику бренда (`ch-top-brands`, `ch-lost-sales`, etc.) заполняет `#filterBrand`.  
Повторный клик — сбрасывает фильтр.

**Debounce**: текстовый поиск (`#filterSearch`) обновляет данные через 250 мс после остановки ввода.

## SKU-таб («Сравнение с конкурентами»)

### Поиск
- Поле `#skuSearch` — ввод артикула Ozon (числовой ID)
- `onSkuInput()` → поиск в `allData` по полю `Артикул Ozon`
- При нахождении: `renderSkuDetail(row)`

### Фильтры конкурентов
- `#skuCountSelect`: сколько конкурентов показать (5/10/15/20)
- `#skuMetricSelect`: метрика сортировки (выручка / продажи)
- `#skuModeSelect`: режим (ближайшие сверху / топ-конкуренты)
- `#skuGroupSelect`: группа сравнения: лидеры категории или похожие SKU. Похожие SKU выбираются из той же категории по той же категории веса или цене за грамм в коридоре ±40%.
- Сохраняются в `localStorage` под ключом `coffeeDashboardSkuFilters`

### Диагностика воронки
- KPI-блок показывает товар, конкурентный бенчмарк и относительное отклонение для показов, CTR, конверсии в корзину, конверсии посещение → заказ и выкупа.
- Для конверсий бенчмарк взвешенный: рассчитывается как отношение суммарных событий к суммарному знаменателю среди сравниваемых SKU.
- Автоматически определяется этап с наибольшим отрицательным отклонением и выводится рекомендация.
- При менее чем 100 показах или 30 посещениях выводится предупреждение о недостаточном объёме данных.

### Графики SKU
| ID | Тип | Что показывает |
|---|---|---|
| `ch-sku-radar` | Radar | 6 метрик выбранного товара vs среднее конкурентов (в %) |
| `ch-sku-funnel` | Funnel | Воронка на 1 000 показов: товар vs взвешенный бенчмарк |
| `ch-sku-stars` | Scatter | Продажи × Конверсия — 4 квадранта |
| `ch-sku-imp-ctr-matrix` | Scatter | Показы × CTR — 4 квадранта, размер точки = выручка |
| `ch-sku-efficiency` | Bar | Индекс эффективности (сводный по 4 метрикам) |
| `ch-sku-metric-cart` | Bar | Конверсия в корзину из карточки |
| `ch-sku-metric-rev` | Bar | Выручка на визит |
| `ch-sku-metric-conv` | Bar | Конверсия в заказ |

## KPI

**Первый ряд** (базовые):
Общая выручка, продано товаров, количество SKU, количество брендов, средняя цена, средний % выкупа, упущенные продажи, средняя доля рекламных расходов.

**Второй ряд** (продвинутые):
Потенциальная выручка, средняя конверсия в заказ, средняя выручка на визит, средняя цена за грамм, доля без дефицита, средние рекламные расходы, высокий потенциал роста (>p75), риск OOS (>30%), эффективных карточек (индекс >80).

## Система справки

- Кнопка `?` на каждой карточке графика → `openChartHelp(id)` → открывает модалку
- Модалка заполняется из `CHART_DEFS[id].help` (поля `shows`, `how`, `why`)
- Закрытие: кнопка «Понятно», крестик, клик по оверлею, Escape

## Примечания

- Все числовые поля маппятся через `NUMERIC_FIELDS` — словарь `{русское_название: ключ}`
- `limitScatter()` ограничивает scatter/bubble до 500 точек (по выручке) для производительности
- `parseNum()` чистит форматированные значения (` ₽`, `X из 28`, `0001-01-01`)
- Графики используют `LinearGradient` для столбцов и палитру `COLORS` (16 цветов)
- Адаптивная вёрстка для мобильных (768px, 480px)
